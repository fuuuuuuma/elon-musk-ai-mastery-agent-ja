# 🎁 Grok 4.5で使える 非エンジニア向けプロンプト100選

Grok 4.5は、文章づくりや調べもの、資料の下ごしらえまで幅広くこなせるAIです。ここでは、非エンジニアの日常の仕事や暮らしでそのまま使える依頼文を100個、10のジャンルに分けて用意しました。

**受け取り方**: 各プロンプトのコピーボタンでコピーするか、このファイルをダウンロードしてください。

**使い方**: 気になるプロンプトをコピーして、Grok 4.5のチャット欄やGrok Build、Claude Codeにそのまま貼り付けてください。文中の[ ]の部分は自分の状況に合わせて書き換えてください。/goal や /loop がついているものは、Claude CodeやGrok Buildのような自律実行に対応した環境で使うと、複数ステップの作業や繰り返しの確認をまとめて任せられます。

## 仕事の文章・メール

### 1. 依頼メールを丁寧に整える
場面: 取引先やお客様に何かをお願いするメールを送る前に

```text
これから送る依頼メールの下書きを見せるので、失礼のない言い回しに整えてください。要件は変えずに、読みやすく短くしてください。
[ここに下書きを貼り付け]
```

### 2. 断りづらい依頼を角が立たない形で断る
場面: 予定が合わない・条件が合わない依頼を丁寧に断りたいとき

```text
以下の依頼を、相手の気分を害さずに断る返信文を作ってください。理由は簡潔にし、可能であれば代案も1つ添えてください。
[依頼内容を貼り付け]
```

### 3. お詫びメールを状況に合わせて作る
場面: 納期遅れや対応ミスをお客様に謝罪するとき

```text
下記の状況について、お客様へのお詫びメールを作ってください。言い訳がましくならないように、事実→謝罪→今後の対応の順でまとめてください。
状況: [起きたことを書く]
```

### 4. 長い連絡事項を箇条書きで整理する
場面: 社内やチームに複数の連絡事項をまとめて送りたいとき

```text
以下のメモを、要点ごとに箇条書きで整理した社内連絡文にしてください。優先度が高いものを先頭に置いてください。
[メモを貼り付け]
```

### 5. 敬語の使い方をチェックしてもらう
場面: 送信前に敬語が正しいか不安なとき

```text
この文章の敬語におかしなところがないか確認してください。おかしい箇所があれば、元の文のニュアンスを保ったまま直してください。
[文章を貼り付け]
```

### 6. 提案文を相手のメリット中心に書き直す
場面: 商品やサービスの提案文を、相手目線に書き直したいとき

```text
以下の提案文を、相手にとってのメリットが最初に伝わるように書き直してください。専門用語は避け、簡単な言葉に置き換えてください。
[提案文を貼り付け]
```

### 7. 長文の返信メールを短くまとめる
場面: 返信が長くなりすぎたとき、要点だけに絞りたい

```text
この返信メールを、要点を落とさずに半分くらいの長さに短縮してください。
[メールを貼り付け]
```

### 8. 催促メールを柔らかい言い方で送る
場面: 返信や支払いなどの催促を、角が立たないように送りたいとき

```text
以下の内容について、相手に催促するメールを作ってください。強い言い方にならないように、確認をお願いする形にしてください。
催促したい内容: [内容を書く]
```

### 9. 複数の候補日をわかりやすく提示する
場面: 打ち合わせの日程調整をメールで送るとき

```text
以下の空いている日時をもとに、相手が返信しやすい日程調整メールを作ってください。候補は箇条書きにしてください。

```

### 10. 文章の口調を相手に合わせて変える
場面: 同じ内容を、社外向け・社内向けなど相手によって書き分けたいとき

```text
以下の文章を、[社外のお客様向け／社内のチーム向け]の口調に書き直してください。内容は変えないでください。
[文章を貼り付け]
```

## 会議・議事録

### 11. 会議メモを議事録の形に整える
場面: 会議中に取ったメモを、あとで正式な議事録にしたいとき

```text
[文字起こしを貼り付け]
```

### 15. 決定事項を関係者に共有する文章を作る
場面: 会議で決まったことを、参加していない人にも伝えたいとき

```text

[決定事項を貼り付け]
```

### 16. 次回会議までの宿題を一覧化する
場面: 会議の終わりに、各自の宿題を整理したいとき

```text
以下の会議内容から、次回までに誰が何をするかの宿題リストを作ってください。期限が明記されていないものは「期限未定」としてください。
[会議内容を貼り付け]
```

### 17. 反対意見をやわらかく伝える言い方を考える
場面: 会議で違う意見を言いたいが、強い言い方にしたくないとき

```text
以下の意見に対して、反対の立場をやわらかく伝える言い方を3パターン考えてください。相手の意見を否定しすぎない言い回しにしてください。
相手の意見: [意見を書く]
```

### 18. 過去の議事録から決定の経緯を探す
場面: 「なぜこの決定になったのか」を過去の記録から調べたいとき

```text
これまでの議事録を渡すので、[知りたいテーマ]について、いつ・どんな議論を経て今の方針になったのか、経緯を時系列でまとめてください。
[議事録をまとめて貼り付け]
```

### 19. 1on1の内容をメモから振り返り用にまとめる
場面: 上司や部下との1on1のメモを、あとで振り返れる形にしたいとき

```text
以下の1on1のメモを、話した内容・気づき・次回までのフォローアップの3項目に整理してください。
[メモを貼り付け]
```

### 20. 定例会議の議事録づくりを毎回サポートしてもらう
場面: 毎週の定例会議の議事録を、いつも同じフォーマットでサッと仕上げたいとき

```text
/loop 7d
定例会議が終わったタイミングで、直近の会議メモを議事録フォーマット(日付・参加者・決定事項・保留事項・次回までのアクション)に整えてください。前回までの議事録と見比べて、同じ話が繰り返されている論点があれば教えてください。
```

```text
3. 私の希望条件([条件を書く])に合いそうな順に並べる
```
